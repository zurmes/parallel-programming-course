## 1. Введение
Целью работы является получение практических навыков работы с реальными многоузловыми вычислительными системами (суперкомпьютерами). В задачи работы входит: изучение удаленного подключения по протоколу SSH, настройка среды окружения Intel Parallel Studio, компиляция MPI-программ под архитектуру Intel Xeon и постановка задач в очередь через планировщик Slurm.

## 2. Теоретические сведения
Работа на высокопроизводительных кластерах отличается от локального запуска необходимостью работы через **планировщик задач (Job Scheduler)**. На кластере «Сергей Королёв» используется планировщик **Slurm**. Пользователь не запускает программу напрямую, а формирует пакетный скрипт (batch script), в котором описывает требуемые ресурсы:
*   Количество физических узлов (`--nodes`).
*   Количество процессов на один узел (`--ntasks-per-node`).
*   Раздел (очередь) исполнения (`--partition`).

Для компиляции на кластере используется специализированный оптимизированный компилятор **Intel C++ Compiler (mpiicpc)**, обеспечивающий максимальную производительность для процессоров семейства Intel Xeon.

## 3. Характеристики системы
*   **Кластер:** «Сергей Королёв» (СГАУ).
*   **Архитектура узлов:** Вычислительные узлы на базе процессоров Intel Xeon.
*   **Среда компиляции:** Intel Parallel Studio XE 2016.
*   **Операционная система:** CentOS Linux.
*   **Планировщик задач:** Slurm.

## 4. Скрипт запуска Slurm (run.sh)
Для постановки задачи в очередь Slurm был подготовлен конфигурационный скрипт, запрашивающий 2 физических вычислительных узла и по 4 процесса на каждом (суммарно 8 MPI-процессов):

```bash
#!/bin/bash
#SBATCH --job-name=Lab5_MPI
#SBATCH --nodes=2
#SBATCH --ntasks-per-node=4
#SBATCH --partition=batch
#SBATCH --output=slurm-%j.out

source /soft/intel/parallel_studio_xe_2016.3.067/bin/psxevars.sh intel64
mpirun ./app_mpi
```

## 5. Описание реализации и запуска
Программа реализует параллельное умножение матриц методом горизонтальной декомпозиции с помощью коллективных операций коммуникации (аналогично Лабораторной работе №3). 
Для проведения испытаний использовался размер матрицы 500x500.

Компиляция на кластере производилась командой:
`mpiicpc -O3 -std=c++11 main.cpp -o app_mpi`

Запуск задачи в очередь:
`sbatch run.sh`
